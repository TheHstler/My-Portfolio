PodPlayer
Transcript-Powered AI Podcast Platform

A full-stack podcast web application where the transcript powers every feature — from clickable timestamps to AI-generated summaries — turning passive listening into active, structured engagement.

🔍 The Problem

Podcast listeners often consume content passively while multitasking, leading to low information retention. Transcripts provided by major platforms are typically raw, unstructured, and inconsistent across sources — making them unreliable for navigation, summarisation, or AI processing.

💡 The Solution

PodPlayer normalises inconsistent transcript data from multiple sources into one consistent structure, then uses that structured data to power every interactive feature — clickable timestamps, entity highlights, AI summaries, and bookmarked notes — all driven directly by the transcript.

⚙️ Tech Stack & Role

Role: Full-Stack Developer
Frontend: React, React Router
Backend: Node.js, Express.js, RESTful API architecture
APIs & AI: Claude API (Anthropic) for summarisation and entity extraction, AssemblyAI for audio transcription with speaker diarization, Taddy API for podcast search and transcript retrieval

🚀 Key Features

  • Transcript Normalisation Layer: Converts inconsistent transcript formats from Taddy and AssemblyAI into a single consistent JSON structure before any processing occurs.
  • AI-Generated Summaries & Insights: Personalised episode summaries and entity extraction powered by Claude, generated directly from the normalised transcript.
  • Interactive Transcript: Clickable timestamps, speaker identification, and highlighted entities (people, places, concepts) with linked information.
  • Bookmarking & Notes: Timestamped notes and AI-generated bookmark explanations, saved to a persistent notes library across episodes.
